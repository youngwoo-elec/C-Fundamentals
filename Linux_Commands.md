# 🐧 Linux/Unix 핵심 명령어 가이드 

 **띄어쓰기에 유의하기**

## 1. 디렉토리(폴더) 관리
- `pwd` (Print Working Directory): 내가 지금 서 있는 폴더의 전체 경로 출력.
- `ls` (List): 현재 폴더의 파일/폴더 목록 보기.
  - `ls -al`: 숨겨진 파일까지 상세히 보기.
- `cd [경로]` (Change Directory): 특정 폴더로 이동.
  - `cd ..`: 한 단계 위(부모) 폴더로 나가기.
  - `cd ~`: 내 홈 디렉토리로 바로 가기.
- `mkdir [이름]` (Make Directory): 새 폴더 만들기.

## 2. 파일 관리
- `touch [파일명]`: 내용이 비어 있는 새 파일 만들기.
- `rm [파일명]` (Remove): 파일 지우기.
  - `rm -rf [폴더명]`: 폴더와 그 안의 모든 내용을 강제로 지우기 (주의!).
- `cp [원본] [복사본]` (Copy): 파일 복사.
- `mv [원본] [이동지]` (Move): 파일 이동 또는 이름 변경.
- `echo "내용"` : 화면에 내용 띄움
- `echo "내용" > [파일명(.txt)]` : 짧은 내용을 한 줄로 파일에 넣음
- `echo "내용" >> [파일명]` : 기존 파일에 내용을 추가

## 3. 화면 및 시스템 제어
- `clear`: 터미널 화면 깨끗하게 비우기.
- `history`: 내가 이전에 입력했던 명령어 기록 보기.
- `&&`: 명령어 두 개를 연속으로 실행 (예: `mkdir test && cd test`).

## 4. 컴파일 및 실행 (C언어 전용)
- `gcc [파일명].c -o [결과파일명]`: C 코드를 기계어로 번역(컴파일).
- `./[결과파일명]`: 컴파일된 실행 파일 실행하기.

## 5. Git 버전 관리 (코드를 서버에 올리기)
- `git status`: 현재 변경된 파일이 무엇인지 상태 확인.
- `git pull origin main --rebase`: (필수!) 작업 시작 전, 서버의 최신 코드를 내 컴퓨터로 가져와 합치기 (push 오류 때도 사용)
- `git add [파일명]`: 변경된 파일을 '장바구니'에 담기. (`git add .`은 모든 변경사항 담기)
- `git commit -m "메시지"`: 장바구니에 담긴 파일들에 '설명(메모)'을 달아 확정하기.
- `git commit --amend -m "새 메세지"`: 커밋을 지우고 새 메세지로 '덮어쓰기' 
- `git push origin main`: 내 컴퓨터의 기록을 깃허브 서버로 전송하기.
